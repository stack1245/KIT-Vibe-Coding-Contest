# 계획

### 개발자 보안 인식 및 교육 개선 프로젝트

**개요**

요즘에 와서 개발자들이 그나마 보안쪽에 신경쓰긴 하지만, 그럼에도 계속해서 보안을 생각하지 않는 사례가 계속 나타나고 있음 이를 개선하기 위해, 직접 현재 코드의 취약점을 분석하고 리포트와 함께 취약점들을 이해하고 나중에 더 잘 고칠수 있게 솔루션 해주는 애플리케이션이 있었으면 좋겠다는 생각으로 프로젝트 기획

- **`기타적으로 추가하면 좋은것들`**
    
    **0-1. LLM은 항상 CTF 라이트업과 원데이 취약점들을 학습하며 보안적인 지식과 익스플로잇 흐름을 계속해서 습득함**
    
    ㄴ 어떻게 습득할껀데?
    
    ㄴ파인 튜닝하지말고 owasp top 10, cve 같은 목록을 시스템 프롬포트 + rag로 넣으면 될거 같습니다
    
    **0-2. 개발자가 누군지 IP, 아이디 등을 통해 감지하고, 계속해서 올린 코드에 동일한 취약점이 존재하는지 등을 분석해 학습 유닛등을 생성함**
    
    ㄴ 어떻게 사용자가 자주 터트리는 취약점이 무엇인지 기억하고 솔루션 할것인가?
    
    (한 LLM 세션이 끝날때 마다 사용자에 관한 정보는 날아감)
    
    ㄴ db에 유저 테이블 만들고 분석할때 간단하게 결과를 저장해놓으면 어떨까요? 이러면 “이런 유형의 취약점을 n번 이상 냈다”라는 형식으로 쿼리 한줄 정도로 저장하면 좋을거 같습니다 다음 분석때 저장된 내용을 확인하고 전이랑 같은 실수를 했을경우 도움을 주는 쪽으로 하면 될거 같네요 기억 문제는 llm 세션이 끝나고 db에 남으니까 문제 없습니다
    
- **`어떤식으로 진행할지`**
    1. **웹으로 개발자가 전체 코드를 올리거나, 깃허브 링크를 올림**
        
        ㄴ 깃허브는 어떻게 받을것인가? 계정 연결? 혹은 직접 들어가서 확인?
        
        ㄴ oauth 연동하는 식으로 하면 될거 같고 전체 코드를 올렸을땐 textarea에 붙여 넣는 방식으로 만들면 될거 같습니다
        
        ㄴ 전체 코드를 받으면 혹시나 생길 악성코드 같은것들은 어떻게 필터링 할것인가?
        
        ㄴ 코드를 실행하느게 아니라 텍스트로 읽기만 한다면 악성코드여도 실행이 되진 않으니 문제가 없을거 같습니다 다만 실습환경에서 코드를 직접 실행하는건 안되고 llm이 별도로 생성한 샌드박스 코드만 돌아갈 수 있게 만들면 될거 같습니다
        
        ㄴ 가이드 만들어 놓기, 악성코드인지 확인, virustotal api 악성코드인지 확인
        
        ㄴ 코드를 어디에 저장하고 분석할것인가? 디스크가 터지는것도 생각해야되지 않나?
        
        ㄴ 코드 전체를 파일로 저장할 이유는 없는거 같습니다 굳이 저장해야하면 /tmp 만들어서 거기에 저장하게 하는 방식으로 했다가 분석 끝나면 삭제 처리하는 걸로 바꾸면 될거 같습니다
        
        ㄴ 일단 분석할때만 /tmp에 넣는다고 해도, 말도안되게 큰 파일이면 서버에 손상이 갈수 있으니, 파일을 서버에 복사하기전, 파일의 크기를 한번 필터링 하면 좋을것 같습니다.
        
        ㄴ 애초에 업로드 단위를 어떻게 잡을 것인가?(단일 파일, 폴더, 프로젝트 전체, 레포 등)
        
        ㄴ 최대는 300kb 정도면 될거 같고 업로드 한계는 30개로 제한 해놓으면 될거 같습니다(폴더(zip,7z등) 안에 있는 파일 포함)
        
        ㄴ 서버에 손상가지 않을 정도로 올라가는것이 목표이고, 에초부터 프로그램 자체가 큰 경우도 있으니, 현재 /tmp에 올라온 파일들의 크기를 보고 유동적으로 파일 리미트를 정하는게 좋을거 같음
        
        ㄴ 가능한 파일 확장자는 어디까지 제한할거임?
        
        ㄴ 파일 확장자는 굳이 특정한것만 제한하는것이 아닌, 차라리 파일을 읽고 LLM이 자동으로 분석해서 이건 안되겠다 하면 떨구고 이건 괜찮겠다 하면 OK하는식이 좋을듯
        
    2. **개발자가 올린 코드를 LLM이 분석한후, 취약점과 보안적으로 코드에 추가하면 좋을점들을 찾음**
    
    ㄴ 만약 코드에 취약점이 없으면 어떻게 할것인가?
    
    ㄴ 취약점이 없어도 코드 상으로 개선이 가능한 부분을 보여주는 식으로 만들면 될거 같습니다
    
    ㄴ 개선할것도 없는 html, css 같은 프론트앤드 파일만 던지거나, 아니면 그냥 프로그램이 아니라 이상한걸 던졌을땐 어떻게 할것인가?
    
    ㄴ 업로드 단계에서 파일을 필터링하면 될거 같습니다 이미지나 바이너리 파일 같은걸 올렸을땐 mime 타입 확인해서 텍스트형 파일 아니면 업로드 안되게 막아놓으면 될 거 같습니다
    
    ㄴ 필터링 방법?
    
    ㄴ LLM 으로 필터링
    
    1. **찾은 취약점을 설명하고 어떤식으로 터지는 취약점인지, 시뮬레이션을 함**
        
        ㄴ 어떻게 시뮬레이션 할꺼임?
        
        ㄴ llm이 공격 시나리오를 단계적으로 생성하면 될거 같습니다 예를들어  sql injection이면 “입력값 → 쿼리 실행 → 인증 우회” 정도의 흐름을 코드 스니펫과 같이 순서대로 보여주면 밑에 4번에서 직접 해볼 수 있으니까 시뮬레이션은 텍스트 + 코드면 충분할거 같네요
        
        ㄴ 어떻게 시나리오 할꺼임?
        
        ㄴ 설명을 어디서 어떻게 나오게 할꺼임?
        
        ㄴ 카드 하나 만들어서 취약점명과 심각도, 행당 코드 위치, 자세한 설명을 분석 결과 페이지에 나타내면 될거 같습니다 이걸 연계해서 실습까지하면 될거 같네요
        
    2. **실제 LLM이 만든 폼을 통해 취약점을 직접 실습하고 취약점에 대해서 이해하도록 함 (실시간으로 LLM이 엔드포인트를 만들어 취약점 실습 페이지를 생성, 삭제함)**
